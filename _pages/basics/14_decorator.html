Chapter 14. Decorator#

Decorator란?#

decorator는 함수 wrapper로 “함수를 감싸서(wrapper) 실행 흐름을 바꾸는 것”이다. 실제 작동 원리는 wrapper가 argument 받는다

Decorator 정의#

함수를 이용한 decorator 정의#

def deco(func):
    def wrapper(*args, **kwargs):
        print("before")
        result = func(*args, **kwargs)
        print("after")
        return result
    return wrapper

@deco
def hello(name):
    print(f"hello {name}")

위의 코드에서 hello()를 실행하면 실제로는 다음 코드가 실행된다.

hello = deco(hello)

클래스를 이용한 decorator 정의#

Decorator 사용#

import datetime 

class DatetimeDecorator:
    def __init__(self, f):
        self.func = f 

    def __call__(self, *args, **kwargs):
        print(datetime.datetime.now())
        self.func(*args, **kwargs)
        print(datetime.datetime.now())

# 사용 
class MainClass:
    @DatetimeDecorator
    def main_function1():
        print("main function1 start")
    
    @DatetimeDecorator
    def main_function2():
        print("main function2 start")

my = MainClass()
my.main_function1()
my.main_function2()

Decorator 사용 사례#

Flask 웹 프레임워크 라우팅#

python에서는 다양한 decorator가 제공된다. Flask 웹 프레임워크에서는 라우팅을 위한 @app.route라는 decorator를 사용한다.

  • @app.route는 URL 경로를 함수에 연결한다. Flask 어플리케이션은 특정 URL에서 요청이 들어올때, 해당 URL에 연결된 함수를 실행할 수 있다.

from flask import Flask 

app = Flask(__name__)

@app.route("/")
def hello():
    return "Hello World!"

로깅(Logging)이 필요할 때#

보통 decorator를 사용하여 함수의 호출과 실행 시간을 로깅한다. 이를 통해 프로그램의 실행 흐름을 추적하거나 성능 문제를 진단하는데 도움이 될 수 있다.

사용자 인증할 때 (Authentication)#

웹 어플리케이션에서 사용자가 특정 기능에 접근하기 전에 인증을 받는 경우가 있다. 이때 decorator를 사용하여 코드의 중복을 줄이고 가독성을 높일 수 있다.

def auth_decorator(func):
    def wrapper(user):
        if user.is_autheticated:
            return func(user)
        else:
            raise PermissionError("로그인이 필요한 기능입니다.")

@auth_decorator
def view_profile(user):
    # 프로필 조회 로직 
    pass
